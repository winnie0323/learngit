三、实验过程

创建learngit空目录，把这个目录变成Git可以管理的仓库，learngit目录下编写一个readme.txt文件， 修改这个文件内容，使用add、commit提交修改之后的文件，学习两者的用途以及区别。 用git log可以查看提交历史、现在的版本状态，用git reflog 可以查看命令历史，用git reset 回到未来某个版本。 然后使用git rm删除文件，添加远程仓库（GitHub），从远程仓库中克隆过来.

四、总结

在撤回修改时，当还没add时用 $ git checkout -- readme.txt 已经add时先用 $git reset HEAD readme.txt 再用 $ git checkout -- readme.txt git add命令实际上就是把要提交的所有修改放到暂存区（Stage）， 然后，执行git commit就可以一次性把暂存区的所有修改提交到分支(master）。 在添加到远程仓库中遇到一些问题：使用SSH时无法推上去，没有权限进行。之后使用了http来push，使用http网址来进行push的时候最开始没有删除已经相连的，导致加不上去，删除之后就可以push成功了。 push到GitHub上面之后，用公网网址进入总是失败，使用了很多方式都没解决，之后通过请教同学，将 html文件重命名成"index"之后发现可以显示了。
